# 🚀 Day 17 - Quiz Game API using FastAPI

> **Part of my #100DaysOfCode Journey**

This project is a part of my journey to **revisit Python fundamentals and backend development concepts** by rebuilding classic Python projects using **FastAPI** and modern backend development practices.

The original Quiz Game was a console-based application. In this version, I redesigned it as a **RESTful API**, exposing quiz functionality through HTTP endpoints with interactive API documentation using **Swagger UI**.

The primary goal of this project was to reinforce backend architecture, API design, request validation, and clean code organization while following industry-standard development practices.

Rather than simply recreating the console application, I redesigned it using a modular backend architecture similar to what is commonly used in production applications.

## Backend Architecture

This project follows a layered architecture by separating responsibilities into different modules:

* **API Layer** – Handles HTTP requests and responses.
* **Schema Layer** – Validates incoming request data.
* **Service Layer** – Contains the business logic.
* **Data Layer** – Stores quiz questions.

This separation improves readability, maintainability, testing, and scalability.
